Plugins

Create a plugin including test-suite, readme with one-liner. hemera-cli
Please ensure that a plugin never extends another plugin that use the same topic name. This results in a plugin which can not be run alone because the publisher is not aware of that composition.

You can create a plugin in two different ways:

Create a plugin for publishing as package


const hp = require('hemera-plugin')
const myPlugin = hp((hemera, opts, done) => {
  const topic = "math"

  hemera.add({
    topic,
    cmd: "add"
  }, function (req, cb) {
    cb(null, {
      result: req.a + req.b
    })
  })

  done()
}, '>=3')

myPlugin[Symbol.for('name')] = 'foo'
myPlugin[Symbol.for('options')] = {}

module.exports = myPlugin
Use always the plugin helper to create new plugins hemera-plugin. It will provide a consistent interface and checks the bare-minimum version of Hemera.

Create a plugin inline


const hp = require('hemera-plugin')
const myPlugin = hp((hemera, opts, done) => {
  const topic = "math"

  hemera.add({
    topic,
    cmd: "add"
  }, function (req, cb) {
    cb(null, {
      result: req.a + req.b
    })
  })

  done()
}, '>=3')

myPlugin[Symbol.for('name')] = 'foo'
myPlugin[Symbol.for('options')] = {}

hemera.use(myPlugin)

Plugin versioning

Plugin dependencies are managed with NPM.

Plugin registration

A plugin must be registered before the ready function is called. The ready function will initialize all plugins. Default plugin options are preserved if you don't overwrite them. 


hemera.use(plugin)

Register multiple plugins at once


hemera.use([plugin, plugin])

Default options.

Pass an object as second arguments to overwrite default options.


hemera.use(plugin, { a: 1 })

After

Calls a function after all the previously defined plugins are loaded, including all their dependencies.


hemera.use(plugin).after(cb)
// or
hemera.use(plugin)
hemera.after(cb)

Payload validators

Registered plugins can have different payload validators. This feature depends on the implementation of the validator. In Hemera-Joi the option payloadValidator will be used to check which validator your plugin used. E.g when you don't set the payloadValidator in the options you can change it later in the corresponding plugin with:


hemera.setOption("payloadValidator", "hemera-joi")

Shared data

Sometimes it is useful to share data across plugins. This data can be accessed from all other plugins. You can also declare other decorators as dependencies.


hemera.decorate("magicNumber", fn, ['dep1', 'dep2'])
